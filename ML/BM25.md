> [!summary]
> BM25 (Best Match 25) is a **probabilistic ranking function** used in information retrieval to score the relevance of documents to a given search query. It is the backbone of most classical keyword-based search engines and remains a strong baseline even in the era of neural IR.

---

# Why introduce BM25 over [[TF-IDF]]
[[TF-IDF]] has some issues:
**Document length is penalized** 
If we consider two documents A,B containing the same term `cat`, with $\text{tf}(\text{cat},A)=1$ and  $\text{tf}(\text{cat},B)=10$ we will have that document B is more relevant according to tf-idf to the query `cat` because 10% of words are cat in document B compared to 1% in document A.

But if we know that A has a length of 10000 words while B has a length of 100, we might be over-weighing the importance of document B since it's harder for a longer document to obtain a high percentage of a word compared to a short document.

This leads to TF-IDF over-penalizing longer documents (to avoid phenomena like keyword stuffing) and under-penalize shorter documents.

To solve this BM25 introduces a hyperparameter to tune the effect of the  document length normalization component.

**Lack of saturation**
TF-IDF scales linearly with TF. But in increase of in the TF component is more important for low values of TF than for higher values: if one document has 2 matching terms while another document has 1 matching term that's a 100% increase while if one document has 101 and another 100 it's only a 1% difference but the linear scaling gives it the same increase in importance.

For this reason we may want to introduce some saturation effect to the tf, leading to diminishing returns in increases in tf.

# BM25's formula
Given a document $D$ and a set of queries $Q$ that we want to use to retrieve, the score of document $D$ is computed as:
$$ \text{BM25}(D, Q) = \sum_{i=1}^{n} \text{IDF}(q_i) \cdot \frac{f(q_i, D) \cdot (k_1 + 1)}{f(q_i, D) + k_1 \cdot \left(1 - b + b \cdot \frac{|D|}{\text{avgdl}}\right)} $$

Where:

| Symbol            | Meaning                                                      |
| ----------------- | ------------------------------------------------------------ |
| $f(q_i, D)$       | Term frequency (absolute frequency) of $q_i$ in document $D$ |
| $\|D\|$           | Length of document $D$ (in words)                            |
| $\text{avgdl}$    | Average document length in the corpus                        |
| $k_1$             | Term frequency saturation parameter (typically `1.2–2.0`)    |
| $b$               | Length normalization parameter (typically `0.75`)            |
| $\text{IDF}(q_i)$ | Inverse Document Frequency of $q_i$                          |
We can see how increasing the hyperparameter $b$ leads to penalizing longer documents more and increasing $k$ reduces the saturation effect.

We can see how setting $b=1$ and $k\rightarrow\infty$ gives us regular tf-idf where tf is the relative frequency.
In the same way setting $b=0$ and $k\rightarrow\infty$ gives us tf-idf without any document length normalization, using raw counts instead of tf.

### Modified IDF Term
Instead of the IDF term used in [[TF-IDF]] we use a modified version 
$$ \text{IDF}(q_i) = \ln\left(\frac{N - \text{df}(q_i) + 0.5}{\text{df}(q_i) + 0.5} + 1\right) $$

| Symbol           | Meaning                              |
| ---------------- | ------------------------------------ |
| $N$              | Total number of documents            |
| $\text{df}(q_i)$ | Number of documents containing $q_i$ |
This modified version comes from a probabilistic theory framework called the Binary Independent Model, while tf-idf was a largely heuristic approach.
This version mathematically represents the log-odds of a term appearing in a relevant document versus a non-relevant document (the 0.5 terms are just to avoid blowup if a term never appears and works as a smoothing constant).
This IDF can actually go negative in the case a term appears in more than half the documents, this is good as it signals how this term is so common we can treat it as noise (e.g. `the`, `a` etc.)

# Usage in conjunction with neural search
Even though BM25 is often  surpassed in performance by ANN search using neural vectors it can be useful to use in conjunction with dense neural vectors using [[Hybrid search for RAG|hybrid search]], for example using [[Reciprocal Rank Fusion]]

# Extensions
There are also a few extensions like BM25+ addressing documents with zero TF, and BM25F addressing documents with multiple fields like HTML, papers etc.

Addresses a flaw where documents with zero TF for a term get no contribution. Adds a lower-bound delta ($\delta$) to the TF component.
